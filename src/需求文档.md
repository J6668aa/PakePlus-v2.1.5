### 美甲工作室预约系统需求文档

---

#### 一、项目概述
开发一个专为美甲师设计的预约管理系统，支持在日历中添加预约信息（时间、项目、客户姓名、电话），并在日历上可视化展示。系统具备开始/结束时间管理功能，以及提前5分钟的声音提醒机制。

---

#### 二、核心功能需求

##### 1. 日历视图
- **显示当前月份**：默认显示当前月份的日历。
- **日期选择**：
  - 支持点击切换上一个月和下一个月。
  - 当前日期高亮显示。
  - 已有预约的日期用不同颜色标记。
- **日历布局**：以周为单位显示，包含星期一至星期日。

##### 2. 预约创建与编辑
- **添加预约**：
  - 点击某一天进入“选择预约时间”界面。
  - 时间段按30分钟间隔划分（如10:00, 10:30, 11:00...）。
  - 每个时间段可选择是否可用（默认不可用，需手动设置为可用）。
  - 用户可选择预约时间并填写以下信息：
    - 客户姓名
    - 客户电话
    - 美甲项目（下拉选择）
    - 开始时间
    - 结束时间（自动计算或手动输入）
- **保存预约**：点击“确认选择”后保存到日历。

##### 3. 日历展示
- **预约显示**：
  - 在日历上以彩色方块形式显示预约项。
  - 方块内显示项目名称缩写（如“法式”、“贴钻”等）。
  - 鼠标悬停时显示详细信息（客户姓名、电话、项目、时间）。
- **点击查看详情**：
  - 点击某个预约项弹出详情窗口。
  - 显示完整信息：客户姓名、电话、项目、开始时间、结束时间。
  - 提供“开始服务”按钮。
  - “结束服务”按钮（点击后标记服务完成）。

##### 4. 提醒功能
- **倒计时提醒**：
  - 当预约开始前5分钟，系统自动播放提示音。
  - 提示音内容：“即将开始服务，请准备。”
  - 提示音可在设置中关闭。
- **视觉提醒**：
  - 接近开始时间时，对应预约项背景变色或闪烁。

##### 5. 数据管理
- **本地存储**：
  - 所有预约数据使用 `localStorage` 存储，确保离线可用。
- **数据结构**：
  ```json
  {
    "appointments": [
      {
        "id": "unique-id",
        "date": "2026-01-18",
        "startTime": "10:00",
        "endTime": "10:30",
        "customerName": "张女士",
        "customerPhone": "13800138000",
        "service": "法式美甲",
        "status": "pending" // pending, started, completed
      }
    ]
  }
  ```

##### 6. UI/UX 设计要求
- **整体风格**：
  - 清新、简洁、女性化。
  - 主色调：粉色系（#FFD1DC）、浅灰背景（#F8F9FA）。
  - 字体：圆润字体（如思源黑体或苹方）。
- **交互细节**：
  - 按钮悬停效果：轻微放大+阴影。
  - 操作反馈：成功保存后显示短暂提示（如“预约已创建”）。
  - 错误处理：输入不合法时给出清晰提示。

##### 7. 技术栈
- **前端技术**：
  - HTML5 + CSS3 + JavaScript（原生）
  - 使用 `localStorage` 实现数据持久化。
  - 使用 `Audio` API 实现声音提醒。
- **响应式设计**：
  - 支持手机端和平板端浏览。
  - 移动端适配：横向滑动切换月份。
